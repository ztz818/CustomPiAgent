# 通用项目审计 Skill

## 使用场景

### 直接调用
- 用户明确要求审计、核查、风险分析时
- 需要对项目进行合规性检查时
- 需要从项目文档中提取审计结论时

### 何时不使用
- ❌ 只是整理文档 → 使用 `audit-workspace-organizer` skill
- ❌ 只是提取文档内容 → 使用对应的文件处理 skill
- ❌ 简单的数据查询 → 直接回答，不需要完整审计流程

## 审计工作流程（十步法）

### 快速索引

1. **了解项目基本情况** - 读取文档，提取核心信息
2. **公开网站核实真实性** - 招投标、立项、建设进度查询
3. **查看项目地址现状** - 地图影像验证
4. **查看项目证件真实性** - 立项、土地、建设手续核查
5. **重复融资核查** - 专项债、债券、质押查询
6. **核实资本金到位及来源** - 到位情况、来源合规性
7. **核实贷款流向和还款来源** - 受托支付、资金流向追踪
8. **项目总投与回款能力分析** - 总投合理性、回款能力
9. **借款主体信用风险评估** - 区域风险、企业财务、负面信息
10. **风险识别与问题发现** - 基于前九步，识别风险

**详细流程**：参见 `workflow/` 目录下的分步指南。

## 审计原则

### 1. 互联网验证强制
- 每个关键信息点必须进行互联网验证
- 不能只看项目文档，必须交叉验证
- 验证结果必须记录在审计报告中

### 2. 证据链完整
- 每个审计发现必须有完整的三重证据链
- 原始材料 + 互联网验证 + 逻辑推理
- 缺一不可

### 3. 风险分级清晰
- 🔴 **高风险**：违法违规、可能造成重大损失
- 🟡 **中风险**：合规性存疑、可能造成一定损失
- 🟢 **低风险**：需要关注、但影响较小

### 4. 处置建议可操作
- **立即处理**：高风险问题
- **限期整改**：中风险问题
- **持续关注**：低风险问题

### 5. 项目特定审计手册优先
- 如果项目目录下有审计手册/核查要点，优先参考
- 审计手册中的要求作为补充审计维度
- 但核心十步法仍然适用

### 6. 审计重点深度参考
- 执行审计时，必须参考 `knowledge/审计重点与风险识别.md`
- 该文档包含完整的风险识别维度和审计要点
- 根据项目类型（城投、PPP、农田、医院等），对照相应章节逐项核查

## 审计报告输出

### 报告结构

**一、审计概述**
1. 审计对象
2. 审计范围
3. 审计方法

**二、项目基本情况**
1. 项目概况
2. 融资情况
3. 运营模式
4. 还款来源

**三、审计发现**

每个问题包含：
1. **问题描述**：清晰描述问题是什么
2. **风险等级**：🔴 高风险 / 🟡 中风险 / 🟢 低风险
3. **三重证据链**：
   - **原始材料**：`文件名+页码/行号`
   - **互联网验证**：搜索关键词、信息来源、验证结果
   - **逻辑推理**：违规依据、风险影响、审计建议

**四、审计结论**
1. 综合风险评级
2. 总体评价
3. 处置建议

**五、附件**
1. 原始材料索引表
2. 互联网验证记录表

**报告模板**：参见 `templates/audit_report_template.md`

## 互联网验证工具清单

### ⚠️ 强制要求：使用 web_search MCP

**所有互联网验证必须通过 `web_search` MCP 工具执行**，不得手动搜索或使用其他方式。

**验证流程**：
1. 使用 `web_search` 搜索关键信息
2. 记录搜索关键词、时间戳、结果来源
3. 将验证结果写入工作底稿
4. 在审计报告中引用验证记录

**示例**：
```python
# 验证项目真实性
web_search(query="[项目名称] 招标公告 site:ggzy.gov.cn")
web_search(query="[项目名称] 立项 site:tzxm.gov.cn")

# 验证企业信息
web_search(query="[企业名称] 隐性债务")
web_search(query="[企业名称] 债务违约")
```
