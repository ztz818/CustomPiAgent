# 通用项目审计 Skill

## 设计理念

这是一个**通用审计框架**，不限定行业和项目类型。

### 核心特点

1. **灵活分析**：一步步审核，不套固定清单
2. **自动适配**：根据项目类型调整审计重点
3. **证据驱动**：三重证据链（原始材料+互联网验证+逻辑推理）
4. **可扩展**：通过 `knowledge/` 目录添加专业知识

## 使用方式

当用户说"审计这个项目"时：
1. 先问清楚审计目标和关注点
2. 读取项目文档，理解项目全貌
3. 一步步分析，识别风险点（不套模板）
4. 对关键信息进行互联网验证
5. 生成审计报告

## 目录结构

```
project-audit/

│   └── audit_report_template.md     # 审计报告模板

    └── 城投融资.md                   # 城投项目专业知识
```

## 扩展方式

### 添加行业专业知识

在 `knowledge/` 目录下创建新的知识库文件：

```markdown
# [行业名称]项目专业知识库

## 适用场景
[说明什么情况下使用这个知识库]

## 核心关注点
[列出该行业审计的关键点]

## 关键验证点
[说明如何验证关键信息]

## 相关政策法规
[列出相关法规，注意标注时效性]
```

### 添加专用脚本

在 `scripts/` 目录下添加数据处理脚本，用于：
- 批量提取结构化数据
- 生成特定格式的工作底稿
- 数据清洗和转换

## 与旧版本的区别

| 旧版本（city-investment-audit） | 新版本（project-audit） |
|-------------------------------|------------------------|
| 限定城投项目 | 适用所有项目类型 |
| 固定审计清单 | 灵活分析判断 |
| 预设风险类别 | 根据项目特点识别风险 |
| 城投专业知识内置 | 专业知识可扩展 |

## 使用建议

1. **首次使用**：先理解项目，不要急于套用模板
2. **专业知识**：如果是特定行业，在 `knowledge/` 添加专业知识库
3. **互联网验证**：关键信息必须外部验证，不能只看项目文档
4. **证据链完整**：每个问题都要有完整的证据支撑
5. **审计重点参考**：执行审计时必须参考 `knowledge/审计重点与风险识别.md`，根据项目类型对照相应章节
